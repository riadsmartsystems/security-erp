# План: Відновлення проекту Security ERP

## Що зламано (коротко)

1. **docker-compose.yml** — містить 20+ сервісів, з них 9 мертві (telegram, n8n, postgres, nats, minio, prometheus, grafana, loki, promtail)
2. **Volume-шляхи** — ведуть на `./security_erp_app` (Flutter-додаток), а треба на `./erpnext/security_erp` (Frappe-додаток)
3. **Назва пакету** — скрізь `security_erp_app`, а реальна назва — `security_erp`
4. **Конфлікт порту 8000** — erpnext-backend і security-api обидва на 8000
5. **scripts/start.sh** — викликає неіснуючий `init-minio.sh`, скрипт падає
6. **.gitignore** — ігнорує `security_erp_app/`
7. **Dockerfile.backend** — не підключений до compose
8. **.env.example** — відсутній
9. **configs/mariadb.cnf** — не вистачає налаштувань InnoDB
10. **permissions.py** — неправильна сигнатура функції
11. **Estimate** — немає полів "Технічне завдання" та "Лід"
12. **Contract** — порожній `pass` замість логіки
13. **Installation Act** — `on_submit` не реєструє обладнання
14. **Print Formats** — відсутні для КП та Акту
15. **Warranty Letter** — DocType відсутній
16. **AI-чернетка кошторису** — не реалізована
17. **CI** — не перевіряє Python-код `security_erp`
18. **deploy.sh** — не перевіряє наявність `.env`

## Порядок виконання

1. Створити `CHANGELOG_FIXES.md`
2. Зберегти `.env` → `.env.backup` (тунель, токени, ключі — все збережеться)
3. A-1, A-2, A-3, A-4 (паралельно)
4. A-5 (заміна docker-compose — залежить від A-3, A-4)
5. A-6, A-7 (виправлення Dockerfile та permissions)
6. B-1…B-10 (бізнес-логіка — файли незалежні)
7. C-1 (видалення мертвих тек)
8. C-2, C-3 (deploy.sh та CI)
9. Перевірка: `docker compose config` + `docker compose build`

## Верифікація

- `docker compose config` — валідація синтаксису compose
- `docker compose config --services | wc -l` — має бути 9
- `docker compose build erpnext-backend` — збірка Dockerfile
- `python -m py_compile` на кожен змінений .py файл
- Валідація JSON для кожного зміненого .json файлу

## Примітка щодо тунелю

Налаштування тунелю (`TUNNEL_SECRET`) збережені в `.env.backup`. Щоб відновити cloudflared:
1. Скопіювати `TUNNEL_SECRET` з `.env.backup` в `.env`
2. Додати сервіс `cloudflared` назад у `docker-compose.yml`
