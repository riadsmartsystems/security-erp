# Перевірка двох пунктів перед Сесією 1

## 1. Назва поля матеріалів на Visit — ⚠️ ЗНАЙДЕНО ПОМИЛКУ В ПЛАНІ

**Frappe field name:** `materials` (visit.json рядок 63-64)
**Push key:** `visit_material` (sync_service.py рядок 49: `"visit_material": {"frappe_field": "materials"}`)

План помилково використовує `'materials'` як push_key у scan_screen.dart. Правильний push_key — `'visit_material'` (той самий ключ, що йде в `ADDITIVE_COLLECTIONS`).

Докази:
- `sync_service.py:49` — `"visit_material": {"frappe_field": "materials", "uuid_field": "client_uuid"}`
- `sync_client.dart:132` — `additive['visit_material']` (pull-side merge)
- `test_s1_sync.py:425` — `"visit_material": [...]` у тесті union-merge
- `test_s1_sync.py:113` — `self.assertIn("visit_material", change.additive)` у тесті pull

**Висновок:** scan_screen.dart має використовувати `'visit_material'`, не `'materials'`.

## 2. Значення `op` у `push_batch` — ✅ ПІДТВЕРДЖЕНО: `'upsert'` / `'delete'`

Знайдено в трьох місцях:
- `schemas/sync.py:38` — коментар: `# "upsert" | "delete"`
- `sync_service.py:186` — `if item.op == "delete"` → tombstoning
- `test_s1_sync.py:155,210,265,371,421` — всі тести використовують `op="upsert"`

Backend приймає будь-яке значення (немає валідації), але конвенція — `'upsert'`/`'delete'`. Клієнтські `'create'`/`'update'` працюють, але не відповідають документації та існуючим тестам.

**Висновок:** Всі screens мають використовувати `op: 'upsert'` (включно з visit_detail_screen.dart).

## Виправлення до промту Сесії 1

### Файл 5: scan_screen.dart

**Було (неправильно):**
```dart
op: 'create',
payload: '{"serial_no":"$code","visit_uuid":"${widget.visitUuid}"}',
```

**Має бути:**
```dart
op: 'upsert',
payload: jsonEncode({
  'scalars': {},
  'additive': {
    'visit_material': [
      {
        '_uuid': clientUuid,
        'serial_no': code,
        'item_name': code,
        'qty': 1,
      },
    ],
  },
}),
```

Також: `doctype: 'Visit'`, `name: widget.visitUuid` (без змін — вже правильно в плані).

### Файл 4: checklist_screen.dart

**Було (неправильно):**
```dart
op: 'update',
doctype: 'ChecklistInstanceItem',
name: item.itemUuid,
payload: '{"checked_by":"${checked ? 'current_user' : ''}"}',
```

**Має бути (один createPendingOp замість двох):**
```dart
op: 'upsert',
doctype: 'Checklist Instance',
name: item.instanceUuid,
payload: jsonEncode({
  'scalars': {'status': 'in_progress'},
  'additive': {
    'checklist_instance_item': [
      {
        '_uuid': item.itemUuid,
        'checked_by': checked ? 'current_user' : null,
      },
    ],
  },
}),
```

### Файл 1: visit_detail_screen.dart

**Було:**
```dart
op: 'update',
payload: '{"status":"$newStatus"}',
```

**Має бути:**
```dart
op: 'upsert',
payload: jsonEncode({
  'scalars': {'status': newStatus},
  'additive': {},
}),
```

### Файли 2,3: camera_screen.dart, voice_note_screen.dart

- `doctype: 'MediaAsset'` → `doctype: 'Media Asset'` (з пробілом)
- `op: 'create'` → `op: 'upsert'`
- payload → `jsonEncode({'scalars': {...}, 'additive': {}})`
- Додати `import 'dart:convert';`

### Тест Сесії 3

- Ключ `'materials'` → `'visit_material'`
- `op="update"` → `op="upsert"` (узгоджено з існуючими тестами)
