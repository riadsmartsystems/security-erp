## Column Mapping

| Source Column | ERPNext Field | Transform |
|---------------|--------------|-----------|
| Повне ім'я | `Customer.customer_name` | strip whitespace |
| Телефон | `Customer.primary_phone` | Split on `,;`, take first, prepend `+` |
| Email | `Customer.primary_email` | strip, validate |
| Ім'я + Прізвище | `Contact.first_name/last_name` | Fallback: split Повне ім'я |
| Директор | `Contact.first_name` | Override Ім'я if present |
| ЄДРПОУ | `Customer.edrpou_code` | Currently 0 rows |
| Примітка + bank fields + source | `Customer.remark` | Concatenate unmappable fields |
| Дисконтна, Знижки, Баланс, Теги | — | Skip (no ERPNext equivalent) |

**Hardcoded**: customer_type="Individual", customer_group="All Customer Groups", territory="All Territories", service_level="Standard"

## Data Cleaning Rules

1. **Phone normalization**: `380XXXXXXXXX` → `+380XXXXXXXXX`. Take first phone from comma-separated list. Skip invalid formats.
2. **Name cleanup**: strip, collapse whitespace, title-case
3. **Dedup**: Check by `primary_phone` (not name — names repeat). Skip if phone exists.
4. **Name collisions**: If customer_name already exists but phone is new → append suffix " (2)", " (3)"
5. **Contact name**: Директор > Ім'я+Прізвище > Ім'я > Повне ім'я

## Implementation Steps

1. Create `migrate_galya_clients.py`:
   - Auto-detect .xls vs .csv input
   - For .xls: convert via `libreoffice --headless --convert-to csv`

   - For each row: clean phone, build Customer doc, create via frappe_create, create linked Contact
   - Log progress every 50 rows, track created/skipped/errors counts
   - Idempotent: skip existing by phone check

2. Modify `migrate_all.py`:

   - Default input: path to Галянтерія XLS file

3. Add `--dry-run` flag for preview without creating records
