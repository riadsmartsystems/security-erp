# План: Імпорт каталогу ВіАТЕК в ERPNext

## Мета
Імпортувати нові товари з XML файлу ВіАТЕК в ERPNext, пропускаючи вже наявні.

## Статус: ⚠️ БЛОКОВАНО — PermissionError 403 Guest

Скрипт `import-viatek-catalog.py` не може створювати записи — ERPNext повертає 403 "Користувач Guest" для всіх POST-запитів.

## Кроки для вирішення

### 1. Перевірити логін через curl
```bash
# Отримати SID
curl -v -X POST http://localhost/api/method/login \
  -H "Content-Type: application/json" \
  -H "Host: erp.localhost" \
  -d '{"usr":"Administrator","pwd":"local_dev_admin_2026"}'

# Перевірити створення Item Group
curl -X POST http://localhost/api/resource/Item%20Group \
  -H "Content-Type: application/json" \
  -H "Host: erp.localhost" \
  -H "Cookie: sid=<отриманий_sid>" \
  -d '{"item_group_name":"Test Import Group","parent_item_group":"All Item Groups"}'
```

### 2. Виправити скрипт або використати frappe_client.py
- Порівняти `_get_sid()` в `import-viatek-catalog.py` з працюючою реалізацією в `scripts/migration/frappe_client.py`
- Можливо проблема в: Host header, cookie path, або таймауті з'єднання

### 3. Запустити імпорт
```bash
FRAPPE_PASSWORD=local_dev_admin_2026 python3 scripts/import-viatek-catalog.py \
  --input "/home/joker/Downloads/Viatek product_info_a_uk.xml" \
  --import-items
```

### 4. Перевірити результат
- Зайти в ERPNext → Stock → Item
- Перевірити кількість імпортованих товарів

## Файли
- `scripts/import-viatek-catalog.py` — основний скрипт імпорту (ПОТРЕБУЄ ФІКСУ АВТОРИЗАЦІЇ)
- `scripts/migration/frappe_client.py` — reference реалізація Frappe API клієнта
- `/home/joker/Downloads/Viatek product_info_a_uk.xml` — XML файл каталогу (12.6 MB, 4912 товарів)
