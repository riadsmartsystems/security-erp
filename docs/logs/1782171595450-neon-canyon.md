## План виправлення

### Етап 1: Відновити відсутні DocType джерела (CRITICAL)

**Задача:** Відновити `.py` та `.json` файли для DocType'ів які мають тільки `__pycache__`.

1. **Декомпілювати `.pyc` → `.py`** для `ai_request_log` та `ai_provider`:
   ```bash
   pip install uncompyle6  # або decompyle3 для Python 3.12+
   uncompyle3 erpnext/security_erp/security_erp/security_erp/doctype/ai_request_log/__pycache__/*.pyc
   ```
2. **Перевірити наявність `.json`** для кожного DocType (field definitions). Якщо `.json` теж відсутній — відновити з Frappe:
   ```bash
   docker compose exec erpnext-backend bench --site erp.localhost export-fixtures
   ```
3. **Перевірити ВСІ DocType'и** наявність `.py` + `.json`:
   ```bash
   find erpnext/security_erp/security_erp/security_erp/doctype/ -maxdepth 1 -type d | while read d; do
     name=$(basename "$d")
     has_py=$(ls "$d"/*.py 2>/dev/null | grep -v __pycache__ | wc -l)
     has_json=$(ls "$d"/*.json 2>/dev/null | wc -l)
     echo "$name: py=$has_py json=$has_json"
   done
   ```
4. **Створити `.py` класи** якщо декомпіляція неможлива (на основі `__pycache__/*.pyc` bytecode аналізу або Frappe DocType JSON).

**Файли:**
- `erpnext/security_erp/security_erp/security_erp/doctype/ai_request_log/ai_request_log.py`
- `erpnext/security_erp/security_erp/security_erp/doctype/ai_request_log/ai_request_log.json`
- `erpnext/security_erp/security_erp/security_erp/doctype/ai_provider/ai_provider.py`
- `erpnext/security_erp/security_erp/security_erp/doctype/ai_provider/ai_provider.json`

### Етап 2: Виправити Flutter тести (HIGH)

**Задача:** Забезпечити компіляцію та проходження Flutter тестів.

1. **`database_test.dart:8`** — додати `NativeDatabase.memory()`:
   ```dart
   db = RiadDatabase.forTesting(NativeDatabase.memory());
   ```

2. **`sync_client_test.dart:543,589`** — замінити прямий виклик `_handleConflict` на публічний API. Варіанти:
   - Змінити `SyncClient._handleConflict` на `@visibleForTesting` публічний метод
   - Або тестувати через `pushPending()` який викликає `_handleConflict` внутрішньо
   - Або використати `dart:mirrors` (не рекомендується для Flutter)

3. **`media_upload_service.dart:82`** — видалити кастомний клас `JsonDecoder` (рядки 82-169), використовувати тільки `dart:convert`'s `JsonDecoder` який вже імпортований.

**Файли:**
- `riad_mobile/test/s2/database_test.dart`
- `riad_mobile/test/s2/sync_client_test.dart`
- `riad_mobile/lib/data/sync/media_upload_service.dart`

## Верифікація після виправлень

1. **DocType джерела:** `ls` кожного DocType каталогу — має бути `.py` + `.json` + `.js` (опціонально)
2. **Flutter:** `cd riad_mobile && flutter test` — всі тести компілюються та проходять
3. **Backend:** `python tests/s1/test_s1_sync.py` + `python tests/a4/test_a4_session.py` — проходять
4. **Next.js:** `cd riad_web && npm test` — всі тести проходять
5. **Whisper:** `docker build -t whisper-test services/whisper/` — збирається без помилок
6. **Lint:** `flake8 services/ --max-line-length=120 && black --check services/`
